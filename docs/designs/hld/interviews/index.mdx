## System Design Interview Format
Use the below steps to help you guide through the system design interview process.
#### Step 1: Requirements clarifications
- Functional requirements
- Non-functional requirements
#### Step 2: Back-of-the-envelope estimation
- Estimation of scale
- Estimation of storage
- Estimation of throughput

#### Step 3: System interface definition
- Defining APIs
- Defining database schema
- Defining event handling

#### Step 4: Defining data model
- Relational DB vs NoSQL
- Schema
- Storage
- Partitioning
- Caching
- Replication
- SQL vs NoSQL

#### Step 5: High-level design
- Draw a block diagram with 5-6 boxes representing core components
- Sketch the main components and connections
- Justify your ideas

#### Step 6: Detailed design
- Dig deeper into 2-3 components
- Discuss and draw component diagrams
- Discuss and draw sequence diagrams
- Address bottlenecks and limitations

#### Step 7: Identifying and resolving bottlenecks
- Discuss and resolve bottlenecks, if any
- Discuss and resolve limitations, if any

## Back of the envelope estimations

### Seconds conversion

| Nano-second | Micro-second | Milli-second |
| ---- | --------- |----|
| 1ns = 10^-9 sec |  1us = 10^-6 sec  | 1ms = 10^-3 sec |

:::tip
- Seconds in a day = 86400 sec = **8.6 x 10^4 sec** = **10 x 10^5 sec**
- Seconds in a month = 30 x 86400 sec = **2.6 x 10^6 sec** ~= **3 x 10^6 sec**
- Seconds in a year = 365 x 86400 sec = **3.15 x 10^7 sec** ~= **3 x 10^7 sec**
:::
